I run three businesses while studying: [Horizn Games](https://play.google.com/store/apps/details?id=com.horizngames.blockpuzzlemaster), content creation (350K+ followers, 40M+ views), and [Zode.ai](https://github.com/tusharv2005/zode.ai). People ask which ones actually make money. Here's the truth.

## The Projects

### 1. Content Creation — 350K+ Followers, 40M+ Views

**Time invested:** 15-20 hours/week  
**Revenue:** $$$ (consistent, monthly)  
**ROI:** High

**What it is:** Faceless social media content across TikTok, Instagram, YouTube Shorts. Monetized through platform revenue, brand deals, and affiliate links.

**The economics:**

- Platform revenue: $2-8 per 1K views (varies by platform, content type)
- Brand deals: $500-5K per campaign (depends on followers, engagement)
- Affiliate: 5-15% commission on sales
- Monthly revenue: Consistent four figures

**Why it works:**

Content compounds. I create once, it generates views (and revenue) for months. A video I made in December is still earning in April.

The model scales: batch create 30 videos in one weekend, schedule them to post daily. That's one month of content from two days of work.

**The downsides:**

- **Algorithm dependency:** Platform changes kill reach overnight
- **Attention treadmill:** Stop posting, revenue drops
- **Copycats:** Easy to replicate, hard to defend
- **Burnout risk:** Content hamster wheel never stops

**What I'm doing:** Systemizing. Templates, batching, VA for editing. Less manual work, same output.

**Verdict:** Keep and scale. High ROI, proven revenue model, systemizable.

### 2. Horizn Games — Mobile Game Studio

**Time invested:** 30-40 hours/week (during development), 5-10 hours/week (post-launch)  
**Revenue:** $ (growing, but not sustainable yet)  
**ROI:** Low initially, improving

**What it is:** Indie game studio. First release: Block Puzzle Master on Google Play. Monetized through ads (AdMob) and in-app purchases.

**The economics:**

- Ad revenue: $5-15 per 1K ad impressions (eCPM varies by country)
- In-app purchases: 2-5% conversion rate on free players
- User acquisition cost: $0.50-2.00 per install (organic + paid ads)
- Monthly revenue: Low three figures (growing)

**Why it's hard:**

Games are hit-driven. 10,000 games launch monthly on Google Play. Getting noticed is brutal. Organic growth is slow. Paid ads are expensive.

First game rarely succeeds. It's a learning project. Second game will be better.

**Why I'm still doing it:**

I love game dev. It's fun. And the upside is asymmetric — one hit pays for ten failures. Plus, I'm learning Unity, C#, mobile optimization, user acquisition. Skills compound.

**The downsides:**

- **High competition:** Competing with studios that have teams and budgets
- **UA costs:** User acquisition is expensive (ads, influencers)
- **Retention is hard:** Players churn fast, have to hook them in seconds
- **Long payback period:** Months to recoup development time

**What I'm doing:** Shipping game #2. Testing different genres, faster iteration, better marketing.

**Verdict:** Keep, but don't expect profit for 6-12 months. This is a long game.

### 3. Zode.ai — Open-Source AI Coding Assistant

**Time invested:** 40-50 hours/week (peak development), 10-15 hours/week (maintenance)  
**Revenue:** $ (none yet, by design)  
**ROI:** Portfolio effect (explained below)

**What it is:** Open-source AI coding agent. Apache-2.0 licensed. Free to use, bring your own API keys.

**The economics:**

- Direct revenue: $0
- Donations/sponsorships: Not yet
- Consulting offers: A few inbound (not pursued yet)
- Job offers: Multiple (not looking, but validating)

**Why no revenue (yet):**

Zode is a long-term play. Open source builds:

- **Credibility:** GitHub stars, community respect
- **Network:** Developers who use it, contribute, share it
- **Proof of skill:** Shows AI engineering, TypeScript, developer tools
- **Opportunities:** Jobs, consulting, partnerships come inbound

Revenue isn't the goal right now. Reputation is.

**The downsides:**

- **No immediate income:** Can't pay bills with GitHub stars
- **Maintenance burden:** Issues, PRs, support requests
- **Competitive pressure:** Big players (GitHub, Cursor) have resources I don't
- **Unclear monetization:** When/how to charge without killing open-source ethos

**What I'm doing:** Building community, improving DX, shipping features. Monetization can wait.

**Verdict:** Keep. Portfolio effect is real. This opens doors money can't.

### 4. Snap AR Lenses — 2.96M+ Plays, 70M+ Views

**Time invested:** 10-15 hours per lens, 5-10 lenses total  
**Revenue:** $ (minimal)  
**ROI:** Low direct, high learning

**What it is:** Interactive AR filters on Snapchat. Built with Lens Studio.

**The economics:**

- Direct revenue: Near zero (Snapchat doesn't pay creators directly)
- Brand deals: Possible, but haven't pursued
- Portfolio value: Shows AR skills, creative work

**Why I did it:**

AR is fun. One lens went viral (2M+ plays). That's addictive. Plus, AR development is valuable — Apple Vision Pro, Meta glasses, spatial computing is coming.

**The downsides:**

- **No clear monetization:** Views don't equal dollars
- **Platform lock-in:** Snapchat controls distribution
- **Unpredictable virality:** Can't manufacture hits

**What I'm doing:** Occasional new lenses, but not a focus. Keeping skills fresh.

**Verdict:** Sunset as revenue source. Keep as portfolio piece and learning playground.

### 5. PetalUI — React Component Library

**Time invested:** 20-30 hours/week (initial build), 5 hours/week (maintenance)  
**Revenue:** $ (none)  
**ROI:** Portfolio effect

**What it is:** Open-source React components. Published to npm, documented, maintained.

**The economics:**

- Direct revenue: $0
- Sponsorships: Not yet
- Freelance leads: A few (high-value projects)

**Why I built it:**

I needed these components for my projects. Decided to open-source them. Now others use them, contribute, and occasionally reach out for custom work.

**The portfolio effect:**

Having a polished, live component library shows:

- **Frontend skills:** React, TypeScript, design systems
- **Developer experience:** API design, documentation
- **Open-source contribution:** Community engagement

This attracts freelance and consulting offers. The components themselves don't make money — the opportunities they create do.

**Verdict:** Keep. Low maintenance, high portfolio value.

### Content → Brand Deals

350K followers opened doors. Brands pay $2-5K per campaign. The audience I built creating free content now generates paid opportunities.

GitHub contributions led to:

- Speaking invitations
- Podcast interviews
- Collaboration offers

You can't buy this. You build it.

## The Framework: How to Decide

When evaluating side projects, ask:

### 1. Does it make money now?

If yes: scale it or systemize it  
If no: does it build toward future revenue?

### 2. Does it create opportunities?

Portfolio projects that open doors are worth more than low-paying gigs

### 3. Am I learning valuable skills?

Skills compound. Every project should teach something marketable

### 4. Can I systemize or scale it?

If it requires constant manual effort, it's a job, not a business

### 5. What's the opportunity cost?

Every hour here is an hour not spent elsewhere. Choose wisely.

## If You're Building Side Projects

**Not all projects need to make money.** Some build skills, some build portfolio, some build network. All of these convert to money eventually — just not directly.

**But you need at least one that pays now.** Bills don't wait for portfolio effects to kick in.

**My rule:**

- 60% time → projects that pay now (content, freelance)
- 30% time → projects that pay later (Zode, PetalUI)
- 10% time → projects for fun (experiments, learning)

This keeps cash flow positive while building for long-term upside.

## Final Thoughts

The projects that make money aren't always the ones you expect.

I thought games would be my main income. They're not (yet). Content creation — which I started as a side thing — became my most reliable revenue stream.

Zode makes zero dollars but opened the most doors.

PetalUI costs me time but led to five-figure consulting projects.

The lesson: **diversify your projects, not just to spread risk, but to capture different types of value — cash now, opportunities later, skills forever.**
