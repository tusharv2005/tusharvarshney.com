I've built a lot of projects over the past few years. Some succeeded, some failed, most taught me something valuable. Here's the complete walkthrough.

## The Active Projects

### 1. Zode.ai (March 2026)

**What it is:** Open-source AI coding assistant for VS Code, JetBrains, and terminal.

**Tech stack:** TypeScript, Node.js, Apache-2.0

**What went right:**

- Built with AI (used Claude to build an AI tool — meta)
- Open-source from day one (transparency builds trust)
- Solved my own problem (best validation)
- Clear positioning vs Copilot/Cursor (control + transparency)

**What went wrong:**

- Marketing is hard (technical product, technical audience)
- Documentation takes forever (still catching up)
- Community building is slow (patience required)

**What I learned:**

- Open source is long-term play (reputation > revenue initially)
- Developer tools need exceptional DX (devs have high standards)
- AI coding is the future (no going back)

**Current status:** Active development, growing community

[View on GitHub](https://github.com/tusharv2005/zode.ai)

### 2. Horizn Games — Block Puzzle Master (January 2026)

**What it is:** Mobile game studio. First release: Block Puzzle Master on Google Play.

**Tech stack:** Unity, C#, Google Play

**What went right:**

- Shipped in 30 days (scope was tight, execution was focused)
- Real users, real feedback (humbling and valuable)
- Learned mobile game dev end-to-end (design, development, deployment)
- Monetization works (ads + IAP generating revenue)

**What went wrong:**

- Marketing is harder than building (organic growth is slow)
- User acquisition costs are high (indie budget, AAA competition)
- First game isn't the hit (expected, but still disappointing)

**What I learned:**

- Game dev is fun but brutal (10,000 games launch monthly)
- Players are honest (5-star or 1-star, no in-between)
- Second game will be better (learning compounds)

**Current status:** Live on Google Play, planning next game

[Play on Google Play](https://play.google.com/store/apps/details?id=com.horizngames.blockpuzzlemaster)

### 3. Content Creation — 350K+ Followers, 40M+ Views

**What it is:** Faceless social media pages across TikTok, Instagram, YouTube Shorts.

**Tech stack:** CapCut, Canva, scheduling tools

**What went right:**

- Scalable (faceless = templatable)
- Multiple platforms (diversified risk)
- Real monetization (brand deals, affiliate, platform revenue)
- Compounding growth (content works while I sleep)

**What went wrong:**

- Burnout is real (content treadmill never stops)
- Algorithm changes hurt (what worked last month doesn't work now)
- Copycats are everywhere (easy to replicate)

**What I learned:**

- Consistency beats quality (ship daily > ship perfect)
- Attention is the game (first 3 seconds decide everything)
- Systems over hustle (batch creation = sustainability)

**Current status:** Active, generating revenue, scaling systems

## What I'd Do Differently

### Start with distribution

I built great products no one knew about. Next time: build audience first, product second.

### Ship faster

Perfectionism killed momentum. MVP > perfect. Always.

### Focus on one thing

I spread thin across too many projects. One successful project > ten mediocre ones.

### Charge earlier

Waited too long to monetize. Paying customers validate faster than free users.

### Document as I build

Retroactive documentation is painful. Write as you code.

### Talk to users more

I assumed I knew what people wanted. I was wrong. Ask, listen, build.

## If You're Building

**Ship more, polish less.** Your first version will be bad. That's fine. Users will tell you what to fix.

**Talk to users early.** Don't build in a vacuum. Ask questions, listen to answers.

**Focus on one thing.** Jack of all trades, master of none is real. Pick one project, make it work.

**Build in public.** Share wins, share losses. The audience you build becomes your unfair advantage.

**Charge money.** Paying customers validate. Free users are nice, but they don't pay bills.
